32 # getproperty it was intended that _pullback falls back to literal_getproperty, but we actually
33 # want the opposite to happen, since Zygote should fall back to recursing into the getproperty
34 # implementation by default. Users still want to define custom adjoints using only
35 # literal_getproperty, though. We can't really have mutually recursive definitions here, so we
36 # now always instrument getproperty as literal_getproperty, no matter whether the second
37 # argument is a literal or not.
